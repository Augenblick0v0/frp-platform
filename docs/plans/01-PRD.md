# FRP 商业化内网穿透平台 PRD

## 1. 产品定位

本项目是一套可私有部署、可商业化运营的 FRP 内网穿透服务平台。平台由服务端、管理后台、用户端、Linux 客户端、Windows 安装包客户端组成。用户注册并兑换套餐后，可以通过客户端创建 TCP、UDP、HTTP、HTTPS 隧道，把本地内网服务暴露到公网。

## 2. 核心目标

- 服务端运行在 Linux Docker 环境。
- 第一版只支持单节点。
- 服务端带管理后台 WebUI。
- 客户端支持 Linux WebUI 和 Windows 安装包。
- 客户端可以创建隧道。
- 支持 TCP、UDP、HTTP、HTTPS 内网穿透。
- TCP/UDP 公网端口由系统从后台端口池自动分配。
- HTTP/HTTPS 通过用户绑定域名访问，不需要带端口。
- HTTPS 支持自动申请 Let's Encrypt 证书。
- Nginx 统一接管 80/443，并按域名反代到对应 FRP 隧道。
- 用户通过邮箱注册，邮箱验证码功能完整，邮件由平台自建邮件服务器发送。
- 后台支持套餐、兑换码、购买链接、用户、隧道、域名、证书、邮件、端口池、流量管理。

## 4. 总体架构

```text
公网访问者
  ↓
Nginx 80/443
  ↓
按 Host 分发
  ├── admin.example.com → 管理后台
  ├── panel.example.com → 用户面板
  └── 用户绑定域名 → frps HTTP vhost
        ↓
      frps
        ↓
      frpc 客户端
        ↓
      用户本地服务
```

服务端 Docker 组件：

```text
frp-platform-server
├── nginx
├── frps
├── api-server
├── admin-web
├── user-web
├── postgres
├── redis
├── mail-server
└── cert-manager
```

客户端组件：

```text
frp-platform-client
├── client-core
├── embedded-frpc
├── local-webui
├── log-manager
├── config-manager
└── package-builder
```

## 6. 隧道功能

### 6.1 TCP 隧道

用户填写：

- 隧道名称。
- 本地地址。
- 本地端口。

系统处理：

- 校验套餐是否允许 TCP。
- 校验 TCP 隧道数量限制。
- 从后台 TCP 端口池中选择空闲端口。
- 生成 frpc 配置。
- 返回公网访问地址。

访问形式：

```text
server.example.com:自动分配端口
```

### 6.2 UDP 隧道

逻辑与 TCP 类似，但从 UDP 端口池分配端口。

### 6.3 HTTP 隧道

用户填写：

- 隧道名称。
- 本地地址。
- 本地端口。
- 绑定域名。

系统提示用户配置：

```text
app.user.com CNAME frp.example.com
```

访问形式：

```text
http://app.user.com
```

无需端口。

### 6.4 HTTPS 隧道

用户填写：

- 隧道名称。
- 本地地址。
- 本地端口。
- 绑定域名。
- 开启 HTTPS。

系统处理：

- 校验套餐是否允许 HTTPS。
- 校验套餐是否允许自动证书。
- 检查 CNAME 是否正确。
- 使用 Let's Encrypt 申请证书。
- 生成 Nginx HTTPS 配置。
- reload Nginx。
- 将请求代理到 frps HTTP vhost。

访问形式：

```text
https://app.user.com
```

## 7. Nginx 规则

Nginx 是唯一公网 HTTP/HTTPS 入口。

职责：

- 监听 80。
- 监听 443。
- 管理后台反代。
- 用户面板反代。
- 用户域名反代。
- ACME HTTP-01 验证路径处理。
- HTTPS 证书加载。
- reload 配置。

HTTP 用户隧道：

```text
Nginx :80 → frps HTTP vhost internal port
```

HTTPS 用户隧道：

```text
Nginx :443 TLS termination → frps HTTP vhost internal port
```

关键要求：

- 转发时必须保留 Host。
- 用户访问域名不需要端口。
- 每个域名必须访问自己的隧道。
- 域名不能被多个用户重复绑定。

## 8. 邮件系统

平台自建邮件服务器，用平台域名发送验证码。

推荐地址：

```text
noreply@yourdomain.com
```

功能：

- SMTP 发信。
- DKIM 签名。
- SPF/DMARC 配置提示。
- 邮件发送日志。
- 测试邮件发送。
- 注册验证码发送。
- 找回密码验证码预留。

验证码规则：

- 6 位数字。
- 10 分钟有效。
- 60 秒内同邮箱不可重复发送。
- 验证码使用后失效。
- 同邮箱每日发送次数限制。
- 同 IP 每日发送次数限制。

## 11. 流量和限速

第一版实现基础流量统计：

- 用户总流量。
- 隧道流量。
- 每日流量。
- 套餐总流量使用率。

- 超流量后禁止启动新隧道。
- 客户端拉取配置时返回超限状态。
- 后台标记用户超限。

- 更精确的限速。
- 流量账单。
